# 2026-02-25 - Context Management and Heartbeat Optimization

## Context Bloat Issue
- **Observation:** The `main` session context reached ~961,000 tokens (96% of the 1M limit).
- **Problem:** Every turn (user message or heartbeat) sends the entire history, hitting Google Gemini 3 Flash TPM limits and causing 503 errors.
- **Diagnosis:** Heartbeats were running in the same `main` session as user chat, preventing the context from truly "pruning" or "sleeping".

## Implemented "Standard Protocol" for Context Reduction
1. **Isolated Heartbeats:** Configured heartbeats to run in a dedicated `heartbeat` session. This prevents periodic system checks from carrying the heavy baggage of main conversation history.
2. **Daily Session Reset:** Enabled an automatic session reset at 04:00 daily to ensure the context window remains fresh.
3. **Aggressive Pruning:** Reduced `contextPruning.ttl` from 30m to 10m to clear the cache faster during periods of inactivity.
4. **Memory-First Architecture:** Reinforced the requirement to move all long-term takeaways to `MEMORY.md` before a session reset occurs.

## 2026-02-25 (Continuation) - System Robustness & Skill Refinement

### Company Analyzer Overhaul
- **Problem:** The skill was brittle, using manual `curl` calls to Moonshot and missing report files.
- **Fix:** Unified `analyze.sh` with `run-framework.sh`.
- **Model Switch:** Switched from Moonshot (Kimi) to **Google Gemini 3 Flash** for internal frameworks.
- **Cache Resilience:** Fixed a bug where caching was disabled, leading to TPM hangs.
- **Deliverables:** Ensured `_FINAL_REPORT.md` is generated automatically.

### Workspace Infrastructure
- **Directory Structure:** Created `assets/outputs/` to house ephemeral data.
- **Documentation:** Added `assets/README.md` to define directory usage.

### Protocol Interlock
- **The "Black Box" Recorder:** Updated `HEARTBEAT.md` with a **Reset Interlock**.
- **Timing:** Every day at **03:00 UTC**, the heartbeat now performs a deep review and captures "Daily Distillations" to `MEMORY.md` before the 04:00 session wipe.

### Lessons Learned
- **Sequential > Parallel (for now):** In the current shell environment, sequential framework execution is more stable for debugging than complex background parallelism.
- **Unified Clients:** Never mix manual `curl` calls with high-level API clients in the same skill; it breaks cost tracking and context management.
