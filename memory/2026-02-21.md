# 2026-02-21 - Company Analyzer Sessions

## Cost Optimization Implemented

Successfully reduced per-analysis costs:
- **Before optimization:** ~$0.98 per analysis (8 frameworks + synthesis)
- **After optimization:** ~$0.18 per analysis (8 frameworks only)

**Changes made:**
1. Stripped verbose data acquisition instructions from prompts
2. Replaced 200-400 token preamble with minimalist 50-80 token headers
3. Removed synthesis framework (~$0.80 savings)
4. Total prompt reduction: ~10,500 → ~3,100 tokens (70% reduction)

## Cost Reality Check

| Analysis | Starting | Ending | Actual Cost |
|----------|----------|--------|-------------|
| Apple (baseline) | $1.27720 | $0.30 | ~$0.98 (with synthesis) |
| Google (no synthesis) | $1.27720 | $1.09502 | ~$0.18 ✓ |
| Meta (no synthesis) | $1.09502 | $0.60616 | **~$0.49** |

**Meta cost variance:** Framework 07 (Business Model) output ~1,400 tokens vs target ~400 tokens. Output length variability is main cost driver.

**Lesson:** Kimi ignores output token limits in prompts. Need stricter constraints or post-generation truncation to hit ~$0.20 target.

## Key Learnings

1. **Output length drives cost** — not input tokens
2. **Kimi ignores token limits** in prompt instructions
3. **Synthesis was expensive** — 800-1,200 output tokens = ~$0.80
4. **8 frameworks without synthesis** = optimal cost/quality balance
5. **New format works** — single message, no fragmentation
